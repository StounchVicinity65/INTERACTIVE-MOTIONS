📏 Understanding Units in Simulations 📏 সিমুলেশনে একক পরিচিতি

🔧 Units Used in These Simulations 🔧 এই সিমুলেশনে ব্যবহৃত এককসমূহ

In these simulations, we use a scaled representation to bridge screen-based animations with real-world physics concepts. Here's how it works:

এই সিমুলেশনগুলিতে, আমরা স্ক্রিন-ভিত্তিক অ্যানিমেশনগুলিকে বাস্তব-বিশ্বের পদার্থবিজ্ঞানের ধারণার সাথে সংযুক্ত করার জন্য একটি স্কেলড উপস্থাপনা ব্যবহার করি।

  • Distance:দূরত্ব: Pixels (px) on screen. Scale: 1 px ≈ 0.00026 m (based on CSS definition — 1 CSS px = 1/96 inch). স্ক্রিনে পিক্সেল (px)। স্কেল: ১ px ≈ ০.০০০২৬ m।
  • Time:সময়: Frames instead of seconds — p5.js updates per frame, simplifying animation logic. At 60 FPS, 1 frame = 1/60 s. সেকেন্ডের পরিবর্তে ফ্রেম — p5.js প্রতি ফ্রেমে আপডেট করে। ৬০ FPS-এ ১ ফ্রেম = ১/৬০ সেকেন্ড।
  • Velocity:বেগ: px/frame on screen → m/s in real world. স্ক্রিনে px/frame → বাস্তবে m/s।
  • Acceleration:ত্বরণ: px/frame² on screen → m/s² in real world. স্ক্রিনে px/frame² → বাস্তবে m/s²।
  • Angular Speed:কৌণিক বেগ: rad/frame on screen → rad/s in real world. স্ক্রিনে rad/frame → বাস্তবে rad/s।
  • Angle:কোণ: Radians or Degrees — remains the same. রেডিয়ান বা ডিগ্রি — একই থাকে।
  • Particle Count:কণার সংখ্যা: Unchanged (dimensionless number). অপরিবর্তিত (মাত্রাহীন সংখ্যা)।

This scaled system maintains accurate visual logic while keeping the animation math simple — learners focus on physical concepts, not unit gymnastics.

এই স্কেলড সিস্টেমটি ভিজ্যুয়াল যুক্তি নির্ভুল রাখে এবং শিক্ষার্থীরা পদার্থবিজ্ঞানের ধারণাগুলিতে মনোযোগ দিতে পারে।

Why the Scale Factor? স্কেল ফ্যাক্টর কেন?

Pixel physical size varies across devices. We anchor to the CSS standard: 1 CSS inch = 96 px, 1 inch ≈ 0.0254 m → 1 px ≈ 0.0254/96 ≈ 0.00026 m. This gives a consistent, device-independent scale.

পিক্সেলের ভৌত আকার ডিভাইসভেদে পরিবর্তিত হয়। আমরা CSS মান ব্যবহার করি: ১ CSS ইঞ্চি = ৯৬ px → ১ px ≈ ০.০০০২৬ m।

Why Frames Instead of Seconds? সেকেন্ডের পরিবর্তে ফ্রেম কেন?

p5.js and most animation engines update once per frame. Using frames as the time unit avoids division at every step. At 60 FPS: 1 frame = 0.0167 s. The unit converter above handles the translation to SI units for you.

p5.js প্রতি ফ্রেমে একবার আপডেট করে। ফ্রেমকে সময়ের একক হিসেবে ব্যবহার করলে প্রতিটি ধাপে ভাগ এড়ানো যায়। ৬০ FPS-এ: ১ ফ্রেম = ০.০১৬৭ সেকেন্ড।

Final Note শেষ কথা

The unit converter shows results in m/s, m/s², and rad/s for SI reference — but inside the simulation these are actually px/frame, px/frame², and rad/frame. The physics relationships are identical; only the units differ.

ইউনিট কনভার্টার SI রেফারেন্সের জন্য m/s, m/s² এবং rad/s দেখায় — কিন্তু সিমুলেশনের ভেতরে এগুলো আসলে px/frame, px/frame² এবং rad/frame।

Quick Conversion Summary দ্রুত রূপান্তর সারসংক্ষেপ

  • Distance:দূরত্ব: m → px m → px
  • Time:সময়: s → frame s → frame
  • Velocity:বেগ: m/s → px/frame m/s → px/frame
  • Acceleration:ত্বরণ: m/s² → px/frame² m/s² → px/frame²
  • Angular Speed:কৌণিক বেগ: rad/s → rad/frame rad/s → rad/frame
  • Angle:কোণ: rad or ° — unchanged rad বা ° — অপরিবর্তিত
  • Particle Count:কণার সংখ্যা: unchanged অপরিবর্তিত
